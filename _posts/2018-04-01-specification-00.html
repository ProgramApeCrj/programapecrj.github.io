1.包的声明和导入的书写

很多时候，包的声明和导入，开发工具（如eclipse），会帮我们自动生成，但是他的书写也是有规律的：

    　
  1. package 包名
  2. Java标准类(java.*)
  3. Java扩充类(javax.*)
  4. 第三方类
  5. 你自己的应用程序的类

举个例子(不同类型的声明用空行隔开,可以给导入类写注释):

package test

import java.util.Date;
import java.util.Enumeration;

import javax.sql.*;

//Apache Xerces
import org.apache.xml.*;

import  mytest.test;

2.关键字的顺序(一般是强制性的)：

    对于一个类的定义来说:
        　
      1. 访问控制修饰符
      2. final或abstract (两者不能共存)
      3. class+类名
      4. extends+继承的类名
      5. implements+实现的接口
        举个例子：
      public abstract class xx extends xx implements xx
        对于一个办法的定义来说:
            　
          1. 访问控制修饰符
          2. static final或final static
          3. abstract(与static final或final static不能共存)
          4. 办法返回类型+办法名+括号中的变量
          5. 抛出的错误
            举个例子：
          public static final String xx() throws xx {
        
}
            对于一个变量的定义来说:
                　
              1. 访问控制修饰符
              2. static final或final static
              3. 属性类型
              4. 属性名称
                举个例子：
              public final static String a = " ";

              2. 办法参数中：逗号＋空格隔开参数
              3. ｆｏｒ循环中：分号＋空格隔开参数
              4. 左大括号前都有一个空格

              6. 运算符两侧都有空格（无论是逻辑运算符还是加减乘除）
