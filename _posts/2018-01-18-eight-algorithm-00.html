2.要涉及的理念？

01.稳定性

通俗地讲呢，就是能保证2个相等的数，在排序后，它们的前后顺序和排序前相同。例如:如果 a[ i ] = a[ j ], a[ i ] 原来在 a[ j ] 前，排序后 a[ i ] 还是要在 a[ j ] 位置前。

为什么要这样呢？举个简单的例子，比如有一批客户订单，已经按购买时间排好序了。但是现在你打算先按价格进行排序，把价格一样的归在一起，然后再按时间先后发货。这时候，如果你写的算法是稳定的，那么排序后价格一样的在一起，但是它们仍然是按照购买时间先后的顺序。

由此可知道稳定性的意义所在 : 两个值虽然相同,但是它们的顺序是有意义的,稳定的算法能保证排序后不改变原来先后顺序，保留原来的意义。

3.可以重复利用的工具办法？

less:判断第一个值是否小于第二个值

{% highlight ruby %} public static boolean less(Comparable v, Comparable w) { return v.compareTo(w) < 0; } {% endhighlight %}

exch:交换两个值的位置

之所以写成下面那样，是因为Java里面没有指针的概念，不能说我传某个数组中的两个值到办法里修改，只能传数组引用和两个值的下标。如果不明白，需要理解Java中值传递和引用传递的基础知识。

{% highlight ruby %} public static void exch(Comparable[] a, int i, int j) { Comparable t = a[i]; a[i] = a[j]; a[j] = t; } {% endhighlight %}

show:打印某次排序后的数组内容

用于生成动态的效果，方便理解算法。

{% highlight ruby %} public static void show(Comparable[] a) { for(int i = 0; i< a.length; i++){ System.out.print(a[i]+" "); } System.out.println(""); } {% endhighlight %}

isSorted:判断一个数组是否有序

这个没上面那三个那么重要，可以写也可以不写。

{% highlight ruby %} public static boolean isSorted(Comparable[] a) { for(int i = 1; i< a.length; i++){ if(less(a[i], a[i-1])) { return false; } } return true; } {% endhighlight %}

将上面的内容放在一个SortUtil中

public class SortUtil {
    public static boolean less(Comparable v, Comparable w) {
        return v.compareTo(w) < 0;
    }
    
    public static void exch(Comparable[] a, int i, int j) {
        Comparable t = a[i];
        a[i] = a[j];
        a[j] = t;
    }
                
    public static void show(Comparable[] a) {
        for(int i = 0; i< a.length; i++){
            System.out.print(a[i]+" ");
        }
        System.out.println("");
    }
    
    public static boolean isSorted(Comparable[] a) {
        for(int i = 1; i< a.length; i++){
         
            if(less(a[i], a[i-1])) {
                return false;
            }
            
        }
        return true;
    }
}
